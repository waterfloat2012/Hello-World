QE project:
1.
OpenClinica QE Project repository:
  https://github.com/OpenClinica/oc-qe

  Fork the project to your GitHub account then clone it to your local machine

  Once the project has been cloned, do a git checkout on the branch "3.x"

2.
Install the Cucumber Eclipse Plug-in

  Github:
  https://github.com/cucumber/cucumber-eclipse
  Plugin-website:
  http://cucumber.github.io/cucumber-eclipse/

3.
  Import the project into Eclipse as a Maven project

4.
  Setup your application-<name>.properties (found in src/test/resources) file to point to your OpenClinica instance
  You can copy the existing "application-qe-3x.properties" file and change it match yours

5. The QE project is a Java based project that uses Selenium WebDriver to test the GUI.
The project is layered into:

    a. Cucumber Feature Files, human readable documents that map requirements/features to automation code,
        a feature file has multiple scenarios, each scenario is made up of different combinations of steps

    b. Steps Classes, Java Classes that perform combinations of methods defined in the service class
          Ex: Create a subject and schedule them for an event

    c. Service Classes, Java Classes that perform different logical user actions 
          (login as user, navigate to a certain page, check an e-mail, etc.)

    d. Page Object Classes, Acts as an interface for interacting with the different web pages,
        used by the service class (to click on web elements, look for text on a page, enter a username, etc.)

        service tomcat start
